# AI App Deployment Template

> Production-ready deployment scaffold for AI-generated web applications.

This template provides a **battle-tested deployment pipeline** for projects generated by ChatGPT, Cursor, Lovable, v0, Bolt, Replit, and similar AI coding tools. It turns "works on my machine" into "running in production."

---

## What's Included

| File | Purpose |
|---|---|
| `Dockerfile` | Multi-stage build optimized for Node.js / Next.js apps |
| `docker-compose.yml` | Full stack: app + Nginx reverse proxy + auto-SSL via Certbot |
| `nginx.conf` | Production Nginx config with gzip, security headers, and SSL |
| `.env.example` | All environment variables documented with defaults |
| `DEPLOYMENT.md` | Step-by-step deployment guide for AWS EC2 / any VPS |
| `.github/workflows/deploy.yml` | GitHub Actions CI/CD: lint → build → deploy on push |

## Quick Start

```bash
# 1. Clone and configure
cp .env.example .env
# Edit .env with your values

# 2. Local development
docker compose up --build

# 3. Production deployment
# See DEPLOYMENT.md for full walkthrough
```

## Supported Stacks

This template is pre-configured for the most common AI-generated app stacks:

- **Next.js** (App Router & Pages Router)
- **React + Vite**
- **Express / Fastify / Hono** (Node.js backends)
- **Python + FastAPI / Flask** (swap Dockerfile stage)

> Adapting to other stacks usually requires only changing the `Dockerfile` build stage.

## Pre-Deployment Checklist

Before deploying any AI-generated project, verify:

- [ ] App runs locally without errors (`npm run dev` / `npm start`)
- [ ] All environment variables are documented in `.env.example`
- [ ] Build completes successfully (`npm run build`)
- [ ] No hardcoded `localhost` URLs in production code
- [ ] Database connection strings use environment variables
- [ ] CORS origins are configured for your production domain
- [ ] API keys and secrets are not committed to the repository

## Common Issues with AI-Generated Code

| Issue | Symptom | Fix |
|---|---|---|
| Hardcoded ports | Works locally, fails in container | Use `process.env.PORT` |
| Missing `.env` | `undefined` errors at runtime | Copy `.env.example` → `.env` |
| Dev-only deps in prod | `MODULE_NOT_FOUND` in container | Check `dependencies` vs `devDependencies` |
| No build step | Deploys source instead of optimized bundle | Add `npm run build` to Dockerfile |
| Relative imports broken | Works in dev server, fails in prod | Check `tsconfig.json` paths |
| CORS blocked | API calls fail from browser | Configure allowed origins |
